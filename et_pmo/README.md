# et_pmo

**Enterprise Transformation – Project Management Office (et_pmo)**

---

## Overview

`et_pmo` is the central hub for managing, tracking, and governing enterprise transformation initiatives. It provides standardised processes, templates, tooling, and documentation to ensure that projects are delivered on time, within scope, and aligned with strategic objectives.

---

## Goals

- Establish a consistent project delivery framework across all transformation workstreams.
- Provide visibility into portfolio status, risks, and dependencies.
- Enable data-driven decision-making through unified reporting.
- Foster collaboration between project teams, stakeholders, and leadership.

---

## Repository Structure

```
et_pmo/
├── README.md              # This file – project overview and getting started guide
├── governance/            # Governance policies, decision frameworks, and RACI matrices
├── templates/             # Reusable project templates (charter, RAID log, status report)
├── playbooks/             # Step-by-step delivery playbooks per project type
├── reporting/             # Dashboard definitions and reporting cadence
└── tools/                 # Scripts and utilities for automating PMO tasks
```

---

## Getting Started

1. **Clone or access the repository**

   ```bash
   git clone https://github.com/your-org/et_pmo.git
   cd et_pmo
   ```

2. **Review the governance framework** in `governance/` to understand how projects are approved and overseen.

3. **Use a template** from `templates/` to kick off a new project or workstream.

4. **Follow the relevant playbook** in `playbooks/` for your project type (e.g., technology rollout, process redesign, org change).
